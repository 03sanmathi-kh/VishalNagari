Problem Statement

Digital Integrity Monitoring for Vishalanagari Tech Parks (Pattern Detection & Log Scanning)

Vishalanagari’s tech parks and public portals generate huge volumes of textual logs — entry records, vendor registrations, project updates, service tickets, and audit trails. Governance teams need an automated engine that scans these text streams to detect repeated fraud patterns, duplicate identities, disguised spam, and unusual access signatures quickly and reliably.

The system must search for suspicious patterns across millions of records and produce high-confidence alerts for governance officers so they can act swiftly. Use string-patterning and fast text-search algorithms (Rabin–Karp, KMP, Boyer–Moore, Z) to build a lightweight, scalable monitoring layer.

Algorithms Used

• Rabin–Karp — Hash-based batch detection of repeated / copied text patterns across logs

• KMP (Knuth–Morris–Pratt) — Linear-time exact pattern matching for structured IDs and signatures

• Boyer–Moore — Fast searching for rare malware / malicious signature strings

• Z-Algorithm — Prefix-similarity & template-detection for spotting cloned or templated fake requests

Efficiency of Each Algorithm

Rabin–Karp: Average Time – O(n + m), allows efficient multiple-pattern checks via rolling hash; Space – O(1) extra per pattern

KMP: Time – O(n + m), Space – O(m) for prefix table — ideal for repeated streaming checks

Boyer–Moore: Average Time – sublinear in practice (skips), Worst-case – O(n + m); Space – O(alphabet)

Z-Algorithm: Time – O(n + m) for combined string preprocessing; Space – O(n) for Z array — useful for similarity detection

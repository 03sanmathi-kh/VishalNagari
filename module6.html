Module 6: Earthquake-Resilient Multi-Storey Buildings

Problem Statement

Vishalanagari requires a robust system to evaluate and enhance the earthquake resilience of multi-storey buildings. The city lies in a region where occasional tectonic activity and soil instability pose structural risks. The goal is to identify weak structural elements, optimize load distribution, and ensure that buildings can withstand seismic shocks while keeping construction cost-effective and sustainable.

Algorithms Used

1. BFS (Breadth-First Search)

Used to traverse each structural layer floor-by-floor to detect weak points, broken load paths, or uneven stress distribution. Perfect for level-order scanning of building frameworks.

2. Dynamic Programming (DP)

Used to compute the minimum reinforcement cost and optimal strengthening strategy by considering multiple building paths and storing sub-problem results to avoid recalculation.

3. Tarjan’s Algorithm (Articulation Points)

Used to find critical structural elements whose failure would compromise the building’s connectivity. Identifies elements that require extra reinforcement for seismic resilience.

Algorithm Efficiency

BFS: Time Complexity = O(V + E), Space = O(V). Efficient for scanning building floors/columns in layered order.

Dynamic Programming: Time Complexity = O(n·W) depending on constraints (like load limits). Best for optimization across multiple reinforcement choices.

Tarjan’s Algorithm: Time Complexity = O(V + E), Space = O(V). Efficient for finding all critical structural elements quickly in a building graph.

Design Techniques Used

1. Base Isolation

A modern earthquake-resistant technique using flexible bearings that allow the building to move safely during seismic waves.

2. Shear Walls

Vertical elements designed to resist lateral forces, stabilizing the building against horizontal shaking.

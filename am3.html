Visitor Vehicle Bypass Routing & Control System

Problem Statement

Visitors entering the smart EV city with petrol/diesel vehicles accidentally enter EV-only zones, causing:

  • Traffic jams
  • Enforcement issues
  • Delays for local EV users
  • Confusion for outsiders

The city needs a system to automatically detect visitor vehicles and guide them via a dedicated bypass road without disturbing internal EV traffic.

Solution

A smart control system that:

  • Detects visitor vehicles at city entry
  • Classifies them as EV / Non-EV
  • Automatically routes outsiders through a bypass-only path
  • Prevents entry into EV roads using smart gates
  • Provides fastest bypass using graph shortest paths

Data Structures Used

  • Graph → Represents city roads + bypass roads
  • Queue → Manages incoming vehicles at checkpoints
  • Hash Table → Stores vehicle type (EV / Non-EV)
  • Priority Queue (Min Heap) → Finds fastest bypass route (used by Dijkstra)

Algorithms Used

  • Dijkstra’s Algorithm → Computes fastest bypass for visitors
  • BFS (optional) → Checks if any entry point is blocked
  • Hashing → Quickly identifies visitor vs resident vehicles

Algorithm Efficiency

  • Dijkstra’s Algorithm: O(E + V log V) — efficient for dynamic bypass routing.
  • BFS: O(V + E) — fast alternative path computation if bypass blocked.
  • Hash Table Lookup: O(1) — quickly identifies visitor vehicle type.
  • Queue: O(1) per insert/remove — manages vehicles at checkpoints efficiently.

Step-by-Step Process

1. Vehicle arrives at entry gate → number plate read.

2. Hash table lookup identifies:

  • EV → allowed inside city
  • Non-EV → redirected to bypass

3. If Non-EV:

  • Dijkstra calculates shortest bypass route
  • Vehicle is sent only on non-interfering path
  • Queue manages vehicles waiting at gate
  • If bypass segment blocked → BFS finds next reachable path
  • Gate opens + navigation shown to visitor

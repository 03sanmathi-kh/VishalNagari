Module 2: Dynamic Inventory Prediction in Steel Industry

Problem Statement

In Vishalanagri’s steel industry, production cannot be decided randomly because scrap availability, electricity costs, and construction demand fluctuate daily. Some days scrap is abundant, while other days it is limited. Electricity costs vary between peak and off-peak hours, impacting production expenses. Construction demand can spike or drop depending on new projects. Producing too much steel leads to storage costs, while producing too little causes revenue loss. The challenge is to plan daily production for the next month to meet expected demand while minimizing total production and storage costs. This requires analyzing resource patterns, forecasting demand, and deciding an optimal schedule that balances cost and supply.

Algorithms Used

1. Sliding Window / Sparse Table: Forecast daily steel demand using historical construction data to predict resource requirements efficiently.

2. Segment Tree / Fenwick Tree: Track scrap availability and electricity cost dynamically, allowing fast queries and updates.

3. Heap / Priority Queue: Allocate production priority to days with abundant scrap or lower electricity cost to optimize daily production.

4. Dynamic Programming (DP): Determine optimal monthly production schedule that minimizes total cost while meeting expected demand and storage constraints.

5. Graph Analysis (Dijkstra / Bellman-Ford): Optional: Model supply chain and find minimum-cost paths from suppliers to factory to market for multi-factory scenarios.

Algorithm Efficiency

  • Sliding Window / Sparse Table: O(n) or O(1) per query — fast demand forecasting over historical data.
  • Segment Tree / Fenwick Tree: O(log n) per update/query — efficient dynamic tracking of scrap and electricity.
  • Heap / Priority Queue: O(log n) per allocation — dynamically prioritizes low-cost production days.
  • Dynamic Programming: O(n * m) — n = days in month, m = production options per day, ensures optimal cost schedule.
  • Graph Algorithms: O(E log V) for Dijkstra, O(VE) for Bellman-Ford — useful for multi-supplier optimization.

Algorithm Flow

Step 1: Historical Demand Data → Sliding Window / Sparse Table → Forecasted Daily Demand
Step 2: Scrap & Electricity Data → Segment Tree / Fenwick Tree → Resource Availability per Day
Step 3: Forecasted Demand + Resource Availability → Heap / Priority Queue → Daily Production Priority
Step 4: Daily Priority + Resource Constraints → Dynamic Programming → Optimized Monthly Production Plan
Step 5: (Optional) Multi-Factory Supply Chain → Graph Algorithms (Dijkstra / Bellman-Ford) → Minimum-Cost Routes
